Writing code is easy. Writing code that can be read, understood, and maintained by others (including your future self) is hard. 

Here are three simple, high-impact rules to instantly elevate the quality of your code.

### 1. Choose Explanatory Variable Names

Avoid generic abbreviations or cryptic single-letter names. A variable name should clearly describe what it contains and why it exists.

```typescript
//  Recommend
const currentDate = new Date();
const currentYear = currentDate.getFullYear();
const userAge = currentYear - user.birthYear;
```
