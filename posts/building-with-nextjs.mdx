Next.js has completely revolutionized how we think about React applications. By introducing the **App Router**, Next.js brings React Server Components (RSC) to the forefront, allowing developers to build faster, more secure, and highly optimized web applications.

### React Server Components by Default

One of the biggest shifts in Next.js is that all components inside the `app` directory are **Server Components** by default. This means they render on the server and send minimal JavaScript to the client.

Here is an example of a simple server-side data fetching component:

```tsx
// src/app/users/page.tsx
interface User {
  id: number;
  name: string;
  email: string;
}

export default async function UsersPage() {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");
  const users: User[] = await res.json();

  return (
    <div className="space-y-4">
      <h1 className="text-2xl font-bold">Users List</h1>
      <ul className="divide-y divide-border">
        {users.map((user) => (
          <li key={user.id} className="py-2">
            <p className="font-medium">{user.name}</p>
            <p className="text-sm text-secondary">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```
