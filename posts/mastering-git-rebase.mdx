A clean Git history is a gift to your future collaborators. Instead of pushing a stream of "fixed typo" or "WIP" commits, you can use `git rebase -i` to rewrite history before merge.

### What is Interactive Rebase?

Interactive rebasing lets you modify commits as they are moved to a new base. You can squash multiple commits into one, edit commit messages, split commits, or reorder them.

To run it for the last 3 commits:

```bash
git rebase -i HEAD~3
```

This opens an editor list showing the commits, allowing you to choose actions for each (e.g. `pick`, `squash`, `reword`).
